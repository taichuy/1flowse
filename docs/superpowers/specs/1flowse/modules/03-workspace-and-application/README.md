- 最后更新：2026-04-14 19:45 CST

## 本模块目标

本模块不是先把完整“应用平台”做完，而是先给后续 `flows` 提供稳定宿主。

当前模块只负责回答四个问题：

- 用户如何在工作台看到、创建和选择 `Application`
- `Application` 作为一等交付单元时，根路由和概览页如何组织
- 进入单个应用后，最小应用内导航骨架是什么
- 用户如何通过唯一主入口进入后续 `agentFlow` 编排

## 当前代码事实

- 当前控制台已有正式壳层与顶层导航：`工作台 / 子系统 / 工具 / 设置 / 个人资料`
- 当前首页仍是工作台正式空态，不是应用列表页
- 当前前后端还没有应用列表、应用创建、应用详情、应用概览和应用内路由
- 权限目录中已经存在 `application` 与 `flow` 资源，说明后续 `Flow` 仍需要明确的 `Application` 宿主层

## 本模块范围

- 工作台中的应用列表
- 创建应用
- 选择应用并进入应用根路由
- 应用概览页
- 应用内最小导航骨架
- 进入编排的单一主入口

## 本轮确认

- `Application` 继续保留为用户直接管理的交付单元，不降级成纯技术中间层。
- `03` 不再承载完整“应用平台”愿景，而是改为 `Flow` 的前置宿主容器模块。
- 工作台首页应从当前正式空态升级为应用列表入口，但不承担复杂分析面板。
- 工作台最小责任固定为：
  - 浏览应用
  - 创建应用
  - 搜索 / 筛选应用
  - 选择应用进入后续编排上下文
- `Application` 创建成功后必须能够形成可进入后续编排的主入口；是否同步创建首个默认 `Flow` 草稿，由 `04` 定义具体 graph 形态。
- 应用根路由先落概览页，概览页只承担：
  - 应用基本信息
  - 当前主编排入口
  - 后续能力入口的占位骨架
- 当前模块的应用内左侧导航骨架只保留后续 `flows` 必需最小集合：
  - `概览`
  - `编排`
- 概览页只保留一个明确主行动：
  - `进入编排`

## 明确不纳入本模块

以下内容不再作为 `03` 前置条件，不允许卡住 `04`：

- 协作者管理
- 调用日志
- 监控报表
- 发布状态汇总
- 应用复制
- 应用删除
- API 发布与文档
- 恢复历史版本

这些内容后续若继续需要，作为独立扩展专题补回，不再混入当前 `03` 的最小闭环。

## 与 04 的边界

- `03` 负责 `Application` 作为宿主容器的入口、上下文和导航骨架。
- `04` 负责 `agentFlow` Studio 本身：
  - 画布
  - 图结构
  - 节点与连线
  - Draft / Version
  - 编排编辑体验

固定规则：

- 没有 `03`，`04` 会同时承担应用容器与编排器职责，边界会失焦。
- `03` 只做容器，不提前吞掉 `04` 的编排语义。

## 当前结论摘要

- `03` 保留模块编号，但模块语义改为 `Flow` 前置容器。
- 本模块只保留后续 `flows` 必需的最小产品闭环。
- 后续开发顺序固定为：先补 `03` 的应用宿主容器，再进入 `04 agentFlow`。
