## 1. 设计目标

让 Codex 凭本文件单独开发出精美且一致的 UI，不需要每次重新发明视觉语法。

**"精美"的操作定义：**
- 高对比、信息密度适宜、状态语义清晰
- 每个元素的存在都有明确理由，没有装饰性冗余
- 工具台质感：稳定、克制、有层次，不是消费级 App，不是营销页

---

## 2. 核心原则（Codex 可机械遵守）

### P1 — 白底高对比工作区
- 背景：`#FFFFFF` 或浅灰底 `#F5F5F5`
- 正文：近黑 `#1A1A1A`
- 辅助说明文字：`#8C8C8C`
- **禁止：** 用 `#999` 灰弱文字作为默认正文；大面积深色背景 + 白字仪表板风格

### P2 — 锐利圆角体系（4-8px，三档）
- Badge、Tag、输入框：`4px`
- 按钮、Drawer 内卡片：`6px`
- 卡片容器、面板、Modal：`8px`
- **禁止：** `16px+` 大圆角

### P3 — 克制阴影分两档
- 卡片悬浮：`0 1px 4px rgba(0,0,0,0.08)`（轻）
- 抽屉/下拉面板：`0 4px 16px rgba(0,0,0,0.12)`（中）
- **禁止：** 多层嵌套阴影、彩色阴影、纯装饰性阴影

### P4 — 主色 + 5 种状态色，其余中性
- 主色：`#1677FF`（Ant Design primary token）
- 状态色仅 5 种（见第 5.3 节）
- 类型标签、分类标签、信息标签：一律中性（`#595959` 文字 + `#F0F0F0` 背景）
- **禁止：** 用颜色区分节点类型或标签分类

### P5 — 工具型排版（固定层级）
- 页面标题 H1：`20px / 600 weight`
- 区块标题 H2：`12px / 600 weight / uppercase / letter-spacing: 0.5px`
- 正文：`14px / 400 weight`
- 辅助说明：`12px / 400 weight / #8C8C8C`
- **禁止：** 超大号标题（32px+）、hero 排版、营销型字体处理

## 3. 壳层与编排层的关系

**结论：同一产品系统，两种表达层，不是两套视觉语言。**

| 维度 | Shell Layer（控制台壳层） | Editor UI Layer（画布层） |
|---|---|---|
| 组件基础 | Ant Design + token 覆盖 | 自有轻组件 |
| 信息密度 | 标准 | 更高（节点更紧凑） |
| 装饰程度 | 低 | 更低 |
| 状态色映射 | **共用同一张表** | **共用同一张表** |
| 字体层级 | 统一 | 统一 |
| 圆角/边框 | 统一 | 统一 |
| 阴影逻辑 | 统一 | 统一 |

**操作含义：** 只维护一份状态色 CSS 变量表，Shell 和 Editor 都从这张表取色，**不允许各自定义一套。**

## 4. 信息层级与页面边界规则

### 4.1 根概览页：严格回到基本信息 + 发布状态 + 进入编排

**是。根概览页严格收敛，允许出现的内容仅此：**

| 内容 | 形式 |
|---|---|
| 应用名称、图标、简介、标签 | 文字 + 图标 + 中性 badge |
| 当前发布状态 | `draft` / `published` 状态 badge |
| 最近活动摘要 | 最近 1 条 run（时间 + 状态），纯文本，非完整列表 |
| 单一主入口 | `进入编排`（primary 按钮，唯一一个 primary CTA） |
| 应用操作区 | 复制、删除、设置（放头部，非正文区） |

**禁止出现（无论以何种理由）：**
- 完整画布（Canvas）
- 完整发布配置表单
- API 文档正文
- 调用日志完整列表
- 设计说明、注释性文案、规则解释文字

### 4.2 四个任务域边界（固定，不可混用）

| 任务域 | 唯一回答的问题 | 入口 |
|---|---|---|
| 编排 | 如何编辑 flow、查看节点、准备发布 | 左导航 `编排` |
| 应用 API | 调用方如何接入当前发布契约 | 左导航 `应用 API` |
| 调用日志 | 最近运行发生了什么，哪一条需要排查 | 左导航 `调用日志` |
| 监控报表 | runtime / state / plugin 是否健康 | 左导航 `监控报表` |

**任何单页只回答一个任务域的问题。**

### 4.3 L0-L3 层级规则

| 层级 | 定义 | 示例 |
|---|---|---|
| L0 | 当前页摘要 + 主入口 | 根概览、编排页状态栏 |
| L1 | 当前上下文内详情展开（不离开当前页） | run row → drawer；node → inspector |
| L2 | 独立任务域完整界面 | 调用日志完整列表、API 文档页 |
| L3 | 明确执行动作 | 发布、重试、导出 |

## 5. 交互规则

### 5.1 按钮规则

| 情况 | 处理方式 |
|---|---|
| 有当前上下文可验证的结果 | 使用 `<button>`，明确 label |
| 没有结果（no-op） | 降级：静态文本 / badge / caption / section label |
| "让页面看起来更丰富"的装饰性 CTA | **禁止放置** |

### 5.2 Drawer 与 Inspector 并存：允许，边界严格固定

**结论：Drawer 与 Inspector 在工作区内允许并存，但边界不可互换，不可新增第三种 L1 模型。**

| 模型 | 触发对象 | 触发动作 | 承载内容 |
|---|---|---|---|
| **Drawer** | 列表型数据行（run row、日志行） | 点击行 | 该条目完整详情，保留列表上下文 |
| **Inspector** | 画布内对象（节点、连线） | 点击 canvas 节点 | 节点输入、输出、状态、职责，页内聚焦 |

**Drawer 最小契约（必须全部满足）：**
1. 关闭态：`display:none` 或 `inert` 属性（不能只是 `transform: translateX` 移出视口）
2. 打开态：`role="dialog"` + `aria-modal="true"`
3. 打开时：焦点移入 drawer 内第一个可聚焦元素
4. 打开期间：Tab 不逃出 drawer
5. 关闭后：焦点回到触发该 drawer 的元素
6. 移动端：全宽 `100vw`，不用侧滑窄抽屉

**Inspector 最小契约（必须全部满足）：**
1. 选中节点 → 侧边栏切换为该节点的 inspector 内容
2. 取消选中 → inspector 收起或回到默认占位状态
3. 不使用 drawer 的容器结构或动画表达 inspector

### 5.3 状态语义字典（唯一一份，Shell 和 Editor 共用）

| 语义 | 颜色值 | 适用上下文 |
|---|---|---|
| `running` | `#722ED1`（紫蓝） | 状态点、badge、节点边框 |
| `healthy` / `success` | `#52C41A`（绿） | 状态点、badge、节点边框 |
| `waiting` | `#FA8C16`（琥珀） | 状态点、badge、节点边框 |
| `failed` | `#F5222D`（红） | 状态点、badge、节点边框 |
| `draft` | `#8C8C8C`（中性灰） | badge |
| `published` | `#1677FF`（主蓝） | badge |
| `selected` | 无填充色，仅 `2px solid #1677FF` outline + `box-shadow: 0 0 0 3px rgba(22,119,255,0.15)` | 任何选中态 |

**补充规则：**
- 节点类型标签（`Trigger / LLM / Tool / State`）：中性 badge（`#595959` / `#F0F0F0`），不用高饱和色
- 同一状态在 list、node badge、inspector、drawer 中颜色一致，不允许重映射
- `selected` 不占用任何状态色，仅 outline

## 6. 移动端降级规则

### 6.1 390px 首屏（强制要求）

首屏折叠线以上必须可见：
1. 应用状态 badge
2. 当前页标题
3. 主入口 / 当前域最小可行动作（不超过 2 个）

**禁止首屏被以下内容占用：**
- 完整侧导航
- 统计卡片堆叠超过 3 行
- 设计说明文案
- 无关导航项

### 6.2 小屏编排页：采用"节点列表降级"策略

**结论：小屏编排页在 `max-width: 768px` 断点下，切换为节点列表模式，不保留横向滚动画布。**

**降级展示内容（按优先级选一）：**
1. **首选：节点列表模式** — 以垂直列表展示 flow 节点序列，每行显示节点名称 + 类型 badge + 运行状态；点击行展开 inspector 式详情面板
2. **备选：只读摘要模式** — 展示 flow 摘要（节点总数、当前发布状态、最近变更时间）+ 提示文案

**无论选哪种，必须配合提示文案：**
> "完整编排功能建议在桌面端使用"（14px 辅助文字，非弹窗，非 modal）

**禁止的降级方式：**
- 保留必须横向拖动才能查看的半成品画布（canvas min-width 造成的 overflow）
- 仅缩小字体 / 压缩间距，把桌面版塞进 390px

### 6.3 其他移动端规则
- 所有 drawer：全宽 `100vw`
- 主卡片：单列堆叠，`width: 100%`
- 导航触达：不依赖 hover，路径不超过 2 次点击
- 点击目标（tap target）：最小 `44x44px`

## 7. 明确禁止项

以下禁止优先级高于所有"看起来合理"的局部判断：

| # | 禁止项 | 原因 |
|---|---|---|
| F1 | 根概览页放完整画布 | 页面边界塌陷 |
| F2 | 根概览页放完整发布配置表单 | 任务域污染 |
| F3 | 根概览页放 API 文档正文 | 任务域污染 |
| F4 | 设计说明 / 注释文案出现在产品 UI | 不是用户该看到的内容 |
| F5 | 用颜色区分节点类型 | 颜色只表达运行状态 |
| F6 | `selected` 使用任何状态色 | 选中态只用 outline |
| F7 | 同类对象有时 drawer、有时 modal、有时跳页 | 破坏 L1 模型一致性 |
| F8 | no-op 按钮保留视觉按钮样式 | 假交互契约 |
| F9 | 小屏保留横向滚动画布 | 不是响应式，是破坏体验 |
| F10 | 新增第三种 L1 详情模型 | 破坏 L1 统一规则 |
| F11 | Shell 和 Editor 各自维护独立状态色 | 必须共用状态字典 |
| F12 | 圆角超过 8px（工具台场景） | 工具台不是消费级 App |
| F13 | 使用与运行状态无关的装饰性彩色 | 破坏状态语义清晰度 |
| F14 | 单页同时回答多个任务域问题 | 信息架构边界塌陷 |
