# Frontend Communication Gate

## Ask First

下列改动先与人沟通，再实现：

| 变更类型 | 先沟通 | 原因 |
| --- | --- | --- |
| 新页面 | Yes | 改变用户可见结构 |
| 新流程 | Yes | 改变任务路径和预期 |
| 交互流调整 | Yes | 改变用户行为契约 |
| 视觉方案调整 | Yes | 容易打碎统一视觉语言 |
| 页面内 AI 协作层能力 | Yes | 属于产品级行为，不只是界面实现 |

## Usually Safe To Do Directly

下列改动通常可直接按 Skill 执行：

| 变更类型 | 通常可直接做 | 前提 |
| --- | --- | --- |
| 局部 bugfix | Yes | 不改变既有页面语法 |
| 同类页面内复用已有组件 | Yes | 不引入新交互规则 |
| 职责收敛 | Yes | 只是把混杂实现拆清楚 |
| 明显重复代码整理 | Yes | 没有过早抽象 |

## Escalate Immediately

出现以下信号时，不要继续闷头写：

- 同类对象要采用不同点击行为
- 需要新增一套按钮样式、弹窗语义或页面层级
- 页面逻辑里开始出现大量协议转换、拼接和条件分发
- 你已经说不清“这段状态到底归页面、组件还是共享层”
