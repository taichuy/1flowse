# Frontend Anti-Patterns

| 反模式 | 风险 | 替代方式 |
| --- | --- | --- |
| 页面 God Component | 一个页面改动牵动所有逻辑 | 按变化原因拆出局部状态、局部视图、协议适配 |
| 状态四处散落 | Bug 难追、行为不稳定 | 先定义状态归属，再决定放置层级 |
| 为单次使用强行抽象 | 抽象先于需求，后续更难改 | 保持内联，等第二个真实场景再抽 |
| 协议拼装和渲染混写 | 前端被接口细节污染 | 先在适配层做转换，再交给视图层 |
| 顺手造新交互语法 | 界面一致性被打碎 | 复用既有交互语法，必要时先找人确认 |
| 同类区域点击行为不一致 | 用户心智混乱 | 把问题上升到结构层处理，不做局部打补丁 |
