## 本轮批判

- 当前 demo 已经可运行，但 `overview` 还在承担“解释为什么要 React 化”和“本轮建议”这类实现说明，偏离了工作区 `L0` 概览页只回答当前状态、发布状态、最近运行摘要和唯一主入口的 recipe。
- 左侧栏同时承载导航、repo 现状、本轮批判、embedded snapshot 和下轮方向，导致桌面端与 `overview` 反复讲同一件事；移动端虽然已经把主内容放前面，但整页仍然过长。
- 移动端缺少靠前的任务域切换入口。当前切页主要依赖后置 sidebar，用户需要滚动较长距离后才能换页，不利于把 demo 当成真实 workspace 浏览。
- 当前 demo 已有 `Drawer` 和 `Inspector` 两种 `L1` 详情模型，这是对的；问题不在容器，而在 `L0` 摘要层没有把“去哪看详情”表达得足够短、足够直接。

### 修正策略

1. 把 `overview` 收回到真正的 `L0`：保留应用头信息、发布/草稿状态、最近运行摘要、embedded snapshot 和唯一主入口。
2. 压缩 sidebar 的职责，只保留身份、导航和一个紧凑的 workspace capsule，不再塞入大段批判/计划/重复现状。
3. 给主内容顶部增加移动端任务域切换条，让用户不用回到底部 sidebar 才能切页。
4. 让 `overview` 的最近运行摘要直接把用户带去 `logs` 并打开对应 `Drawer`，形成清晰的 `L0 -> L2 -> L1` 路径。
