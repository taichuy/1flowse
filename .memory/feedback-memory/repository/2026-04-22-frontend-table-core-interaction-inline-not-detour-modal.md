# 后台表格的核心配置交互应内联到对应列，避免绕到额外管理弹窗

## 时间

`2026-04-22 08`

## 规则

- 当后台表格里已经有对象的核心列，例如版本列、状态列或主配置列时，优先把同主题交互直接内联在该列里，不要再额外拆出“管理”按钮和独立管理弹窗。
- 如果当前值已经通过控件本身清楚显示，例如下拉框当前选中的版本号，就不要再叠加“当前”标签、禁用态按钮或重复说明。
- 在同一行里，交互应保持紧凑，优先使用“当前值选择器 + 辅助动作按钮”的组合，例如“版本选择 + 更新”。
- 配置类入口文案应贴近用户任务语义；当动作实质是进入该对象的配置域时，优先使用“配置”，不要用实现味更重的“查看实例”。

## 原因

- 后台表格的主目标是让用户在数据旁边直接完成动作；如果还要绕到额外“版本管理”按钮和弹窗，会打断操作路径，也会把本来很窄的交互做重。
- 当前值已经显示时，再叠加“当前”标签或禁用态，会造成语义重复，视觉上也更臃肿、更难看。
- 动作文案应该反映用户想做什么，而不是底层对象是什么；“配置”更接近用户心智，“查看实例”更像实现细节。

## 适用场景

- 设置页、控制台页、管理台页中的表格行操作设计。
- 某个对象已经有明显的主属性列，例如版本、环境、状态、来源、配置方案。
- 需要在紧凑表格里同时承载查看、切换、升级、配置等核心动作。
