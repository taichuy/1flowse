# 节点变量列表使用单行变量样式和抓手拖拽

规则：节点详情里的自定义变量保存后，应和系统变量列表保持一致的单行结构，不要渲染成两行信息卡或把编辑、上移、下移、删除拆成单独操作行。

原因：变量列表的主任务是扫描和引用变量，系统变量已经建立了 `变量名 + 类型` 的视觉模式；自定义变量如果使用不同布局，会让同类对象看起来不是同一类。

适用场景：`agent-flow` 节点详情中展示自定义输入字段、输出字段、变量或可排序变量列表时，行首使用拖拽抓手做上下排序，行中展示变量名和类型，行尾只放删除等必要动作；编辑可以通过点击变量主体进入浮层。
