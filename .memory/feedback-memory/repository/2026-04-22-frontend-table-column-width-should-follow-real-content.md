# 后台表格列宽应跟随真实内容和控件宽度一起收窄

## 时间

`2026-04-22 09`

## 规则

- 当表格某一列承载的是短文案按钮组或紧凑控件时，列宽应跟随真实内容一起收窄，不要只缩小控件本身而保留原列宽。
- “版本”这类列如果已经把选择器缩小，列本身也应同步收窄，避免选择器右侧留下大块空白。
- “操作”这类列只需要容纳当前按钮组的实际宽度；如果当前只有“配置 / 添加 / 删除”这类短按钮，就不应保留远大于三按钮宽度的列宽。
- 收窄时仍要保留内容最小可读宽度，不能把文案、箭头或按钮挤坏。

## 原因

- 只缩控件不缩列宽，会让表格视觉重心错位，产生明显无意义空白。
- 紧凑表格里，列宽本身就是信息密度的一部分；多余空白会削弱整体紧凑感。
- 列宽跟随真实内容收敛后，说明列和相邻列的空间分配会更自然，也更符合后台管理界面的扫读节奏。

## 适用场景

- 设置页、控制台页、管理台页中的表格列宽微调。
- 某列承载下拉、短标签、短按钮组等紧凑元素。
- 用户明确指出某列右侧或内部存在明显空白，需要继续压缩布局。
